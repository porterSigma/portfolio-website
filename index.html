Memory Manager

A custom C++ memory management system designed to efficiently handle frequent allocations of small objects of varying sizes, a scenario typical in game engines where performance is critical and memory fragmentation must be minimized.

The system features a global MemoryManager that delegates requests to a SmallObjectAllocator, which uses fast bitwise operations to select the correct FixedAllocator. Memory is organized into chunks and blocks, enabling rapid reuse and minimizing fragmentation.

  • Global MemoryManager with SmallObjectAllocator delegation
  • Fast bitwise operation for FixedAllocator selection
  • Chunk and block organization for memory efficiency
  • Convenient macros (MM_NEW, MM_DELETE)
  • Automatic fallback to system allocator for large objects

Ghost'n Castle

Ghost'n Castle is a 2D mobile game inspired by the classic arcade title Pang. The player controls a ghost moving horizontally across the screen, casting spells to defeat enemies.

Each enemy, once hit, splits into smaller ones until fully eliminated. The project was developed in Unity with a focus on the mobile touch controls, scalable difficulty management, and efficient object pooling for optimized performance on Android devices.

One of the project's goals was to explore AI as a method of graphic and sound innovation, experimenting with procedural techniques to create unique visual and audio elements.

AI Soccer System

Development of an advanced AI system for an existing 2D arcade soccer game, completed in collaboration with one other developer. The project scope was strictly focused on designing and implementing intelligent agent behaviors within an established codebase, requiring deep analysis of existing game mechanics including physics, player controls, and field interactions.

The AI system implements a sophisticated role-based architecture where agents dynamically assign themselves roles based on game state analysis. Each role executes specialized behaviors through hierarchical finite state machines, with transition logic that responds to ball possession changes, field positioning, and teammate status.

Technical Implementation

  • Role-based State Machine Architecture: Implemented dynamic role assignment system
  • Predictive Shot & Pass Evaluation: Probabilistic scoring system analyzing angle, distance, and obstacle interference
  • Best Supporting Spot (BSS): Field sampling system identifying optimal positioning for pass reception
  • Ball Repositioning System: Attract-based ball control allowing strategic maneuvering while avoiding opponents and boundaries

Overdrill

Team-based academic project combining Tower Defense and Twin-Stick Shooter mechanics in Unreal Engine. Set in a sci-fi universe where players defend automated drilling operations against waves of strategic alien forces. The game emphasizes tactical decision-making and resource management through alternating combat and construction phases.

The project emphasizes dynamic combat encounters where enemy coordination and tactical decision-making create emergent gameplay scenarios. Different alien archetypes (Melee, Tank, Ranged, Infiltrator, Support) execute complementary strategies, requiring players to balance direct combat with strategic structure placement.
